# References — Índice de Referências Visuais

Coleção de referências visuais para análise do senso estético de Victor Costa.
Alimenta o `SOUL.md` na raiz do repositório.

## Como usar

1. Salve imagens na pasta de contexto adequada (veja categorias abaixo)
2. Opcionalmente crie um `.md` homônimo com metadados (fonte, tags, mood)
3. Compartilhe o caminho no chat para análise — a IA atualiza o `SOUL.md`

## Categorias

| Pasta | Contexto de aplicação |
|---|---|
| `saas/` | UIs de produto, dashboards, SaaS B2B/B2C, apps |
| `portfolio/` | Portfólios de designers, estúdios, agências |
| `branding/` | Identidade visual, logos, sistemas de marca |
| `editorial/` | Layouts editoriais, revistas, livros, publicações |
| `motion/` | Motion design, animação, vídeo, frames de after effects |
| `client/` | Referências coletadas para projetos externos de clientes |
| `misc/` | Referências que cruzam categorias ou não se encaixam nas acima |

## Metadados opcionais (por imagem)

```markdown
---
source: url-ou-nome-da-fonte
tags: [dark-theme, typography, hierarchy, motion]
context: [saas, portfolio]
mood: confident, minimal, technical
---
Notas livres aqui.
```

## Imagens sem metadados

Análise ainda funciona, mas perde contexto de aplicação. Mover arquivos de `misc/` para a pasta correta sempre que possível.
